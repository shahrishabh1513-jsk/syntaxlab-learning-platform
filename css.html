🎨 1. CSS Introduction Foundation

CSS (Cascading Style Sheets) is the language we use to style HTML elements. It controls colors, fonts, layouts, and responsive design.

What makes CSS powerful?
  • Separates content (HTML) from presentation (CSS)
  • One CSS file can style an entire website
  • Enables responsive designs for all devices
  • Saves time and makes maintenance easy
/* CSS Example - Styling a webpage */
body {
    background-color: lightblue;  /* Page background */
}

h1 {
    color: white;                 /* Heading color */
    text-align: center;          /* Center alignment */
}

p {
    font-family: verdana;        /* Font style */
    font-size: 20px;             /* Text size */
}

Try It Yourself:

Create a style.css file and link it to your HTML. Change the body background to your favorite color!

Why "Cascading"?

"Cascading" refers to the order of priority when multiple CSS rules apply to the same element. Styles cascade from top to bottom, with later rules overriding earlier ones unless specificity rules apply.

📝 2. CSS Syntax Structure

A CSS rule consists of a selector and a declaration block.

Syntax Structure: selector { property: value; } 
/* Selector (targets HTML elements) */
p {
    color: red;           /* Property: value */
    text-align: center;   /* Multiple declarations */
}
Key Points:
  • Selectors point to HTML elements
  • Declaration block = { }
  • Each declaration ends with semicolon (;)
  • Property and value separated by colon (:)
Common Mistakes:
  • Missing semicolons between declarations
  • Forgetting closing curly brace }
  • Misspelled property names

🎯 3. CSS Selectors Targeting Elements

Selectors "find" HTML elements to style. There are 5 main categories:

Selector Type Example Description
Element p { } Selects all <p> elements
ID (#) #header { } Selects element with id="header" (unique)
Class (.) .card { } Selects all elements with class="card"
Universal (*) * { } Selects ALL elements
Grouping (,) h1, h2, h3 { } Selects multiple elements 
/* Element selector */
p { color: blue; }

/* ID selector (unique) */
#intro { font-size: 20px; }

/* Class selector (reusable) */
.highlight { background: yellow; }

/* Multiple classes */
<p class="highlight large">Text</p>

Create a class called "btn" that gives buttons a blue background, white text, padding, and rounded corners!

📁 4. CSS Types Inline | Internal | External

External CSS (Best Practice) 
<link rel="stylesheet" href="styles.css">

✓ Reusable across pages
✓ Clean HTML
✓ Faster loading (cached)

Internal CSS 
<style>
  body { background: #f0f0f0; }
</style>

✓ Single page styling
✓ No extra HTTP requests

Inline CSS (Highest Priority) 
<p style="color: red;">Text</p>

✓ Quick testing
✗ Hard to maintain
✗ Not reusable

Priority Order (Highest to Lowest): Inline CSS → Internal CSS → External CSS → Browser Defaults

📏 7. CSS Units px | % | em | rem | vh | vw

Unit Type Description Best For
px Absolute Pixels (1/96 inch) Borders, small elements
% Relative Percentage of parent Responsive widths
em Relative Relative to parent font-size Typography within components
rem Relative Relative to root font-size Accessible typography (PREFERRED)
vh Viewport 1% of viewport height Fullscreen sections
vw Viewport 1% of viewport width Responsive text scaling 
html { font-size: 16px; }  /* 1rem = 16px */
h1 { font-size: 2rem; }    /* 32px */
.container { width: 80%; } /* 80% of parent width */
.hero { height: 100vh; }   /* Full viewport height */
Accessibility Tip: Use rem for font sizes so users can adjust text size in browser settings!

📦 8. CSS Box Model Every element is a box!

The Box Model consists of:
  • Content - The actual content (text/images)
  • Padding - Space inside the border
  • Border - The edge around padding
  • Margin - Space outside the border
.box {
    width: 300px;
    padding: 20px;   /* Adds space INSIDE */
    border: 2px solid black;
    margin: 10px;    /* Adds space OUTSIDE */
}

⬅️ 10. CSS Margin Space Outside

/* All four sides */
margin: 10px;

/* Top/Bottom Left/Right */
margin: 10px 20px;

/* Top Right Bottom Left (clockwise) */
margin: 10px 20px 30px 40px;

/* Individual sides */
margin-top: 10px;
margin-right: 20px;
margin-bottom: 30px;
margin-left: 40px;

/* Auto centering */
margin: 0 auto;  /* Centers block elements */
Margin collapses vertically - space between elements = larger margin, not sum!

📏 11. CSS Padding Space Inside

/* All four sides */
padding: 10px;

/* Top/Bottom Left/Right */
padding: 10px 20px;

/* Top Right Bottom Left */
padding: 10px 20px 30px 40px;

/* Individual sides (unlike margin, padding NEVER collapses) */
padding-top: 10px;
padding-right: 20px;
padding-bottom: 30px;
padding-left: 40px;
Padding increases the element's total size, so adjust width accordingly!

🔲 13. CSS Outline Outside Border

/* Outline doesn't affect layout */
outline: 2px solid red;
outline-width: 3px;
outline-style: dotted;
outline-color: blue;
outline-offset: 5px;  /* Space between border and outline */
Outline is NOT part of the box model - it doesn't take space or affect layout!

📦 14. CSS Box-Sizing Life-Changing!

/* Default - content-box */
.box {
    width: 300px;
    padding: 20px;    /* Total width = 340px! */
}

/* Better - border-box (RECOMMENDED) */
.box {
    box-sizing: border-box;
    width: 300px;
    padding: 20px;    /* Total width = 300px */
}

/* Reset for all elements */
*,
*::before,
*::after {
    box-sizing: border-box;
}
Always use border-box! It makes layout predictable!

⚡ 34. CSS Combinators Relationships

⚖️ 48. CSS Specificity Which Rule Wins?

Selector Type Specificity Score Example
Inline style 1000 style="color:red"
ID selector 0100 #header
Class/pseudo-class 0010 .btn, :hover
Element selector 0001 p, div
Universal (*) 0000 * 
/* More specific = higher priority */
p { color: black; }           /* Score: 1 */
.text { color: blue; }        /* Score: 10 (wins) */
#main p { color: red; }       /* Score: 101 (wins more) */

/* Same specificity? Later rule wins */
p { color: red; }
p { color: blue; }  /* Wins */

⚠️ 49. CSS !important Nuclear Option

/* Overrides EVERYTHING */
p {
    color: red !important;
}

/* Even inline styles won't override */
<p style="color: blue">This text is red due to !important</p>

/* Only another !important can override (with higher specificity) */
#main p {
    color: green !important;  /* Wins if more specific */
}
Use !important SPARINGLY! It breaks natural cascade and makes CSS hard to debug. Only use for utility classes or overriding third-party code.
